## Question Generation Rules

Follow this 4-step process:

1. Generate 6~7 core questions matching the session type (strictly follow session-specific rules)
2. Customize each question to reflect the business summary + target persona
3. Apply style adjustments:
   - "Deep" → strengthen follow-up hints with deeper probing
   - "Soft" → increase empathy expressions in icebreakers
   - "Concise" → reduce to 6 total questions
4. Remove duplicates & optimize order (broad → specific)

## Question Count Guidelines

- Session 1 (Problem Interview): 6~7 core questions (excluding icebreakers)
- Session 2 (Solution Interview): 6~8 questions
- Session 3 (MVP Validation): 6~8 questions

## Section Distribution Rules (REQUIRED)

Every icebreaker and question MUST be assigned to a section.

- icebreaker section: exactly 3 (placed in the "icebreakers" array)
- context section: 1~2 questions
- problems section: 2~3 questions
- alternatives section: 1~2 questions

(Session 1 does NOT include a wtp section — wtp is covered in Session 4)

## Banned Question Patterns (NEVER use these in Session 1)

- "What would need to be different for you to..." (hypothetical future)
- "What would it be worth to you if..." (pricing anchor)
- "What would need to be true for you to..." (hypothetical condition)
- "If you had a magic wand..." or any Magic Wand style questions
- "How often do you..." (frequency estimate — use past event anchor instead)
- "What's your current approach to..." (generic — use specific past event instead)
- Any question starting with "Would you..." or "Could you imagine..."
- Any hypothetical "what if this existed?" scenario
- Any question about price, value, willingness to pay, or budget

## Required Question Patterns (ALL questions must use these forms)

Every question must be anchored to a specific past event or concrete experience:
- "Tell me about the last time you..."
- "Walk me through what happened when..."
- "Think back to a recent situation where... What did you do?"
- "Describe a specific moment when..."
- "What did you actually do the last time...?"
- "Can you give me a specific example of when...?"

## Icebreaker Rules
- Always generate exactly 3 icebreakers
- All 3 must flow naturally as one opening monologue
- First: warm thank you + tension relief (intent: "rapport_building")
- Second: empathize with persona's current situation (intent: "role_context")
- Third: gently explain interview purpose and create psychological safety (intent: "rapport_building")
- Each icebreaker: 2~3 sentences max
- Never mention: solutions, products, or repeat formal greetings
- All icebreakers: section = "icebreaker", report_bucket = "context"

Example (good):
- "Thank you so much for taking the time today. Please feel free to speak openly."
- "I imagine things have been quite busy with [persona situation]. I've heard similar stories from many others."
- "There are no right or wrong answers here. The more honest you are, the more helpful it is. Shall we begin?"

## Required Components Per Question

Each question must include:

1. Question text (past-behavior form ONLY — see Required Question Patterns above)
2. Why this question? (1~3 sentences, linked to lean customer development principles)
3. 2~3 follow-up hints (short prompts that dig deeper into the specific story — plain strings, not objects)

Follow-up hints must be specific probing prompts, e.g.:
- "What did you do right after that happened?"
- "How did that make you feel in that moment?"
- "What was the impact on your work/team/business?"
